The problem arises because the value retrieved from the prompt function is always returned as a string, even if the user enters a numeric value. Therefore, performing mathematical operations directly on this string will result in string concatenation rather than numerical operations.

To solve this problem, we need to convert the user input from a string to a numeric data type (such as an integer or float) before performing the mathematical operations. We use parseFloat() to convert the string to a numeric value. Additionally, we check if the input is a valid number using isNaN(). If the input is not a valid number, an error message is printed to the console. Otherwise, the mathematical operations are performed on the numeric value.
